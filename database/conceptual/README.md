# Conceptual Database Model

This directory contains the **conceptual database model**.

It represents the **domain concepts**, without concern for technology, DBMS, or physical implementation.  
The focus is understanding the business before thinking about tables.

## What you will find here
- Domain entities
- Entities relationships
- Conceptual diagrams
- DBMS - independent representations

## What should NOT be here
- Specific data types
- SQL scripts

## Purpose of this directory
To support learning the conceptual phase of data modeling, documenting the reasoning process and the evolution of domain understanding throughout the project.
